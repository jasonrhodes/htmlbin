Demo note — two patterns shown together on every row, for comparison only.

(A) Split button + caret. Left side edits in the creation surface; caret reveals Edit in Discover / Builder / Agent. Builder is disabled (with a tooltip) when the rule wasn't authored in Builder.

(B) ⋯ actions menu with discrete edit options. The three surfaces are top items in the same popover as Clone / Enable / Disable / Delete, separated by a divider. No row-level primary "Edit" button — every edit is one extra click into the menu. Same Builder disabled-state behavior.

In production you'd pick one of these, not both.
